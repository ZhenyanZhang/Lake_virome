# Host Prediction

Contigs generated by MEGAHIT (v1.2.9) were used to recover metagenome-assembled genomes (MAGs) from lake samples using MetaBAT2 (v2.12.1). After quality control and dereplication (using dRep v3.4.5 with a 95% sequence identity cut-off), we recovered 3,973 non-redundant MAGs meeting the medium or higher quality criteria of the MIMAG standard (completeness ≥ 50%, contamination ≤ 10%, and quality score ≥ 50). These MAGs were taxonomically assigned by GTDB-Tk (v2.4.0) with default parameters. 

*(Note: The specific commands and scripts for assembly, binning, quality control, and dereplication are detailed in a separate chapter.)*

By adding these recovered MAGs and their GTDB-Tk taxonomic annotations into the default iPHoP database (version: Jun_2025_pub), we created a custom database for host prediction. Finally, iPHoP (v1.4.1) was utilized to predict the hosts of the identified lake viruses (vOTUs). A priority-based rule was subsequently applied to the iPHoP output to determine the final host: direct sequence matches (BLAST or CRISPR) were prioritized, followed by the integrated machine learning model predictions based on the highest iPHoP-RF score.

## 1. Consolidate GTDB-Tk Taxonomic Annotations

Prepare the GTDB-Tk output files (both bacterial and archaeal) by consolidating the tree, alignment, and inference results into a single directory required by iPHoP for database construction.

```bash
# Create target directories
mkdir -p 7.host_prediction/drep_bins_tree_all/align
mkdir -p 7.host_prediction/drep_bins_tree_all/infer

# Copy bacterial GTDB-Tk results
cp 3.MAG/drep_bins_tree_bac/gtdbtk.bac120.decorated.tree-table 7.host_prediction/drep_bins_tree_all/
cp 3.MAG/drep_bins_tree_bac/gtdbtk.bac120.decorated.tree 7.host_prediction/drep_bins_tree_all/
cp -r 3.MAG/drep_bins_tree_bac/align/* 7.host_prediction/drep_bins_tree_all/align/
cp -r 3.MAG/drep_bins_tree_bac/infer/* 7.host_prediction/drep_bins_tree_all/infer/

# Copy archaeal GTDB-Tk results
cp 3.MAG/drep_bins_tree_arc/gtdbtk.ar53.decorated.tree-table 7.host_prediction/drep_bins_tree_all/
cp 3.MAG/drep_bins_tree_arc/gtdbtk.ar53.decorated.tree 7.host_prediction/drep_bins_tree_all/
cp 3.MAG/drep_bins_tree_arc/align/* 7.host_prediction/drep_bins_tree_all/align/
cp 3.MAG/drep_bins_tree_arc/infer/gtdbtk.ar53* 7.host_prediction/drep_bins_tree_all/infer/
```

## 2. Create Custom iPHoP Database

Integrate the dereplicated MAGs and their corresponding GTDB-Tk taxonomy into the downloaded iPHoP database.

```bash
iphop add_to_db \
    --fna_dir 3.MAG/bin_drep \
    --gtdb_dir 7.host_prediction/drep_bins_tree_all \
    --num_threads 64 \
    --db_dir ./db/Jun_2025_pub_rw \
    --out_dir ./db/iphop_db_custom
```

## 3. Host Prediction with iPHoP

Predict the bacterial and archaeal hosts for the identified vOTUs using the newly created custom iPHoP database. 

```bash
# Run iPHoP prediction in the background
nohup iphop predict \
    --fa_file 4.vibrant_results/vOTUs.fna \
    --db_dir ./db/iphop_db_custom \
    --out_dir 7.host_prediction/iPHoP \
    -t 64 \
    --min_score 75 > 7.host_prediction/iphop_predict.log 2>&1 &
```
